## 0. 使用范围

本 Skill 用于：

- 高中物理课本例题改编；
- “一题三变”；
- 同步练习改编；
- 实验题改编；
- 知识卡片配题；
- 校本题库建设；
- 教研稿与 Word 成稿；
- 题图、表格、坐标图与规范解析制作；
- 网站题库结构化入库。

# 执行总流程

1. 执行 Skill 0：确认改编定位；
2. 执行 Skill 1：识别物理量、数字、函数名、单位和角标；
3. 执行 Skill 2：规范上下角标；
4. 执行 Skill 3：标准化单位；
5. 执行 Skill 4：区分物理语义层、数学结构层、字体层和 Word 输出层；
6. 执行 Skill 5：应用速度字体专用规则；
7. 执行 Skill 6：分离物理量与单位；
8. 执行 Skill 7：阻止 LaTeX 源码直接进入 Word；
9. 执行 Skill 8：完成题干、图表、解析和答案一致性校验；
10. 执行 Skill 9：将公式中的数学除法统一转为横线分式；
11. 执行 Skill 10：按【题目】【答案】【解析】结构化输出；
12. 删除无用占位符、框体和嘈杂要素；
13. 生成 Word；
14. 渲染检查；
15. 发现异常必须返修后再交付。

# Skill 0：试题改编定位

## 0.1 核心模型不变

必须保留：核心物理规律、关键受力关系、适用条件、原题主要解题路径、本课时知识边界和能力目标。

允许改变：数值、局部条件、情境、设问顺序、信息呈现方式、图像、表格或实验背景。

不得为了追求新颖而改变题目本质。

## 0.2 “一题三变”固定结构

### 一、基础变式——同模型巩固

- 保持原模型和基本解题路径；
- 主要改变数值、局部条件或设问；
- 难度与原题相当；
- 重点考查基础规律和基本计算。

### 二、情境变式——新情境迁移

- 将同一规律迁移到新的生活、实验、生产或工程情境；
- 不改变核心模型；
- 重点考查模型识别和方法迁移；
- 难度与原题相当或略高。

### 三、素养提升——多表征综合

可综合使用文字、图像、数据表格、函数关系、实验数据、误差分析、临界判断、安全评价和方案比较。

要求：难度略高于原题，不超出本课时知识范围，重点考查建模、推理、迁移和综合应用。

# Skill 1：物理量识别

## 1.1 物理量

一般物理量使用 Times New Roman 斜体；速度符号按 Skill 5 单独处理；中文说明性字符使用正体。

## 1.2 数字

数字使用 Times New Roman 正体，不随周围中文字体变化；负号使用数学负号。

## 1.3 单位

单位使用 Times New Roman 正体，不斜体、不加复数、不继承物理量字体。

## 1.4 函数名

sin、cos、tan、ln、arccos 等函数名使用 Times New Roman 正体，函数名与角度或变量之间不人为添加空格。

正确：sin37°、cos53°、tanθ。

## 1.5 说明性角标

F合、m总、v初、E测、E真等说明性文字使用正体。

# Skill 2：上下角标规范

- v₀、x₁、a₂、10⁻³等必须使用真实上下标；
- 变量型角标使用斜体；
- 序号角标、说明性角标和单位指数使用正体；
- Word 中 m/s² 的 2 必须是真上标，不得写成 m/s2；
- 角标不得漂移、拉伸、重叠或成为独立大字号节点。

# Skill 3：单位标准化

## 3.1 单位统一采用斜杠制

正确：m/s、m/s²、m²/s²、N/kg、J/kg、W/m²、C/kg。

不得使用：m·s⁻¹、m·s⁻²、m·s−1、m·s−2、N·kg⁻¹、m s^-2、m/s2。

## 3.2 强制转换规则

- m·s⁻¹、m·s−1、m·s-1 → m/s
- m·s⁻²、m·s−2、m·s-2 → m/s²
- m²·s⁻² → m²/s²
- N·kg⁻¹ → N/kg
- J·kg⁻¹ → J/kg
- W·m⁻² → W/m²

## 3.3 数值与单位

数值与单位之间保留一个规范间距，如 2.0 N、5.0 m/s、10 m/s²。

## 3.4 单位换算

涉及单位换算时，在换算步骤中明确写出单位，例如 54 km/h = 15 m/s。

## 3.5 代数过程中的单位书写

代数计算过程中，不要求每一个代入量都附带单位。先代入纯数值，将整个数值运算部分用括号括起，并在括号后统一标注单位，再写最终结果及单位。

推荐：

v₁ = v₀ + a₁t₁ = (15 + 0.5 × 8) m/s = 19 m/s

不推荐：

v₁ = v₀ + a₁t₁ = 15 + 0.5 × 8 = 19 m/s

规范要求：先列物理公式；再代入纯数值；数值运算整体加括号；括号后统一标注单位；最终结果再次写单位。

# Skill 4：公式分层

## 4.1 物理语义层

识别物理量、数字、单位、函数、角标、运算符、方向和正负号。

## 4.2 数学结构层

生成横线分式、真上标、真下标、根号、括号、函数结构、分段函数、求和结构和矢量符号。

## 4.3 字体字形层

- 速度：Book Antiqua 斜体；
- 其他物理量：Times New Roman 斜体；
- 数字、单位、函数：Times New Roman 正体；
- 说明性角标：正体。

## 4.4 Word 输出层

最终必须使用 OMML 可编辑公式；不得输出 LaTeX 源码；不得用普通文本模拟横线分式；不得用图片代替公式。

# Skill 5：速度字体规则

速度符号及其变量部分统一使用 Book Antiqua 斜体，如 v、v₀、v₁、Δv、vₓ。数字角标、说明性角标使用正体，单位 m/s 使用 Times New Roman 正体。

# Skill 6：物理量与单位分离

- 物理量使用斜体；速度使用 Book Antiqua，其他物理量使用 Times New Roman；
- 单位使用 Times New Roman 正体和斜杠制；
- 示例：a = 2.0 m/s²，F = 10 N，v = 5.0 m/s。

# Skill 7：LaTeX 禁用规则

LaTeX 只可作为中间输入语言，不得直接作为最终 Word 内容。页面出现 \\mathrm、\\frac、\\text、^{-1}、\\sqrt、花括号源码或反斜杠命令，均判定为转换失败。

最终 Word 必须使用 OMML，分式、上下角标、根号和函数结构均可编辑。

# Skill 8：一致性校验

## 8.1 内容一致

题干数据与图表一致，图表与解析一致，解析与答案一致，条件足以得到唯一结果或边界明确的结果，三道变式不得串题。

## 8.2 物理一致

研究对象、过程划分、方向、正负号、适用条件、量纲、有效数字和实际物理意义均正确。

## 8.3 规范解析顺序

1. 说明研究对象或物理过程；
2. 使用“根据……可得”写明物理依据；
3. 列出公式或方程；
4. 代入纯数值，将数值运算整体置于括号内，并在括号后统一标注单位；
5. 写出结论、方向和单位。

常用表述：根据牛顿第二定律可得；根据速度—时间公式可得；根据位移—时间公式可得；根据速度—位移公式可得；根据机械能守恒定律可得；根据动量守恒定律可得；根据库仑定律可得；根据万有引力定律可得；根据闭合电路欧姆定律可得；根据法拉第电磁感应定律可得；根据几何关系可得；根据图像斜率的物理意义可得；根据题意及临界条件可得。

## 8.4 图表一致

坐标刻度、图线斜率、截距、图例、表格数据、题图连接关系以及图中物理量和单位均与题目一致。

## 8.5 渲染一致

最终必须渲染 Word 页面，检查公式间距、字体替换、上下角标、图片清晰度、表格分页、页面溢出、LaTeX 残留、答案框残留和旧单位写法残留。

# Skill 9：除法横线分式规则

数学公式中的除法统一使用横线分式，如 a = F/m、ρ = m/V、v = s/t、k = Δa/ΔF、F = kq₁q₂/r²，在 Word 中均须显示为横线分式。

注意：数学除法使用横线分式；单位采用斜杠制。公式中的 F/m 使用横线分式，单位 m/s 保持斜杠形式。

# Skill 10：结构化输出与网站适配

## 10.1 固定三级结构

每一道变式必须按【题目】【答案】【解析】顺序输出，三者均使用普通正文段落。

## 10.2 禁止框体包裹

【题目】【答案】【解析】不得放入表格、单元格、文本框、形状、底纹框、边框、整段卡片或浮动对象，不得使用绿色答案框或浅色底纹答案框。

## 10.3 表格允许范围

只有题目本身确需呈现实验数据、时刻—速度数据、测量记录、统计数据或方案对比数据时，才允许使用表格；数据表不得包裹整道题目、答案或解析。

## 10.4 删除无用占位符

必须删除：图略、此处插图、待补充、占位图、示意图位置、请在此处插入图片、空白图片框、空白文本框、空表格、空形状和无内容边框。

## 10.5 删除嘈杂要素

必须删除编辑器控制柄、下拉箭头、加号按钮、选中状态边框、重复标题、重复答案、重复图注、乱码、残缺公式、孤立符号、无关英文、无关背景、图标、标签、水印、色块和装饰。

## 10.6 网站结构化要求

最终 Word 中应能直接识别：【题目】后的内容为题干，【答案】后的内容为最终答案，【解析】后的内容为完整解析。不得把答案混在解析末尾或将答案放入表格。

## 10.7 终检

检查是否严格使用【题目】【答案】【解析】；是否仍有答案框、绿色底纹、边框或表格包裹；是否仍有无用占位符；是否仍有 m·s⁻¹、m·s⁻² 等旧单位；代数过程是否采用“(纯数值运算) 单位 = 结果 单位”；公式前是否写明依据；是否存在乱码、空框或嘈杂要素。

# 公式与正文间距强制规则

- sin37°、cos37°、tanθ 不人为加空格；
- 数值、等号、加减号、乘号之间不人为加入多余空格；
- 避免字符扩展、强制字号、强制基线、多余前后间距、重复函数节点、空文本节点和单位指数独立大字号节点；
- 重点检查 sin37°、cos37°、tanθ、m/s、m/s²、N/kg、连续等式、连续乘法、上下角标和横线分式。

# 图像、坐标图与表格规则

## 题图

结构符合真实物理逻辑；受力方向正确；绳线、导线、器材连接正确；物体位置符合题意；图像清晰；不添加无关英文、背景、装饰或提前给出的答案。

## 情景图

只保留建模所需对象和环境，不自动添加数据、公式、受力箭头、结论、速度或加速度标签。

## 坐标图

横纵轴正方向带箭头；物理量和单位靠近对应箭头；物理量斜体、单位正体；刻度、截距、斜率与题目一致；时间刻度位于横轴下方，刻度线直接落在坐标轴上。

## 表格

仅用于真实数据；表头物理量斜体、单位正体；上下角标真实；小数位和有效数字一致；跨页重复标题行；不使用表格包裹题目、答案或解析。

10. 主要公式前写明物理依据；
